## Why API Keys?

<CardGrid stagger>
  <Card title="Simple Authentication" icon="key">
    Not everything speaks OAuth2. CLI tools, IoT devices, scripts, and legacy systems often just need a simple API key.
  </Card>
  <Card title="Keycloak as Authority" icon="shield">
    Keycloak validates every request. No local JWT validation, no token expiry headaches. The key is valid until you revoke it.
  </Card>
  <Card title="Per-Client Prefixes" icon="tag">
    Each client gets its own prefix (`myapp_xxx`). Know at a glance which application a key belongs to.
  </Card>
  <Card title="Full Audit Trail" icon="document">
    Every key creation, usage, and revocation is logged. See who created what and when it was last used.
  </Card>
</CardGrid>

## How It Works

```
┌─────────┐         ┌──────────────┐         ┌──────────────┐
│ Client  │──────▶  │  Your API    │──────▶  │  Keycloak    │
│         │  API    │  (middleware)│  token  │  (validates) │
│         │  Key    │              │  intro  │              │
└─────────┘         └──────────────┘         └──────────────┘
```

1. Client sends request with `X-API-Key` header
2. Middleware calls Keycloak's token introspection endpoint
3. Keycloak validates the key and returns the associated client/scopes
4. Your API processes the request with full identity context

## Quick Example

```typescript
import express from 'express';
import { apiKeyAuth } from '@emdzej/keycloak-api-keys-node';

const app = express();

app.use(apiKeyAuth({
  keycloakUrl: 'https://auth.example.com',
  realm: 'my-realm',
  clientId: 'my-api',
  clientSecret: process.env.CLIENT_SECRET,
}));

app.get('/api/data', (req, res) => {
  // req.apiKey contains validated key info
  res.json({ message: 'Authenticated!' });
});
```
